Welcome to one of the most powerful, yet dangerous, areas of Java development: **Multithreading**.

In a world of multi-core processors, a single-threaded application is like a massive factory with only one worker. Multithreading allows us to hire more "workers" (threads) to perform tasks concurrently, making our applications more responsive and significantly faster.

---

## 1. What is a Thread?

A **Thread** is the smallest unit of execution within a process. While a Process has its own memory space, all threads within that process share the same memory (the Heap), but they keep their own local variables (the Stack).

### Two Ways to Create a Thread:

1. **Extending the `Thread` class:** Simple, but less flexible because Java doesn't support multiple inheritance.
2. **Implementing the `Runnable` interface:** The professional standard. It keeps your task logic separate from the thread management.

```java
// The Professional Way
Runnable task = () -> {
    System.out.println("Worker thread is running: " + Thread.currentThread().getName());
};

Thread thread = new Thread(task);
thread.start(); // NEVER call run() directly; start() creates the new call stack.

```

---

## 2. The Lifecycle of a Thread

Understanding the states of a thread is crucial for debugging deadlocks and performance bottlenecks.

* **New:** Created but not started.
* **Runnable:** Ready to run and waiting for the CPU scheduler to pick it.
* **Running:** Currently executing.
* **Blocked/Waiting:** Waiting for a lock or a signal from another thread.
* **Terminated:** Finished its execution.

---

## 3. Synchronization: The "Traffic Light"

Because threads share the same memory, they can try to update the same variable at the same time. This leads to a **Race Condition**, where the data becomes inconsistent.

To fix this, we use the `synchronized` keyword. It ensures that only **one thread** can enter a block of code at a time by acquiring a "Monitor Lock."

```java
public class Counter {
    private int count = 0;

    // Without synchronized, two threads could read '5' and both update it to '6'
    public synchronized void increment() {
        count++;
    }
}

```

---

## 4. Thread Coordination: `wait()` and `notify()`

Sometimes, threads need to talk to each other. For example, a "Consumer" thread shouldn't try to take data from a queue if it's empty—it should wait until a "Producer" thread adds something.

* **`wait()`**: Tells the current thread to give up the lock and go to sleep.
* **`notify()`**: Wakes up a single thread that is waiting on that object's lock.

---

## 5. The "Senior Dev" Warning: Common Pitfalls

1. **Deadlocks:** Thread A is waiting for a lock held by Thread B, while Thread B is waiting for a lock held by Thread A. They stay frozen forever.
2. **Starvation:** A low-priority thread never gets CPU time because high-priority threads are hogging it.
3. **Overhead:** Creating too many threads can actually slow down your system due to "Context Switching"—the CPU spends more time switching between workers than actually doing work.

---

## 6. The Evolution: `ExecutorService`

In modern Java, we rarely create `new Thread()` manually. Instead, we use a **Thread Pool** via the `ExecutorService`. This reuses existing workers instead of constantly hiring and firing new ones.

```java
ExecutorService executor = Executors.newFixedThreadPool(10);
executor.submit(() -> System.out.println("Task handled by pool"));
executor.shutdown();

```

## Summary Table: Multithreading Concepts

| Concept | Purpose |
| --- | --- |
| **`start()`** | Allocates a new call stack and begins execution. |
| **`join()`** | Makes one thread wait until another thread finishes. |
| **`volatile`** | Ensures a variable is always read from/written to main memory (not CPU cache). |
| **`synchronized`** | Prevents multiple threads from accessing a code block simultaneously. |

### Tip:

When dealing with simple shared counters or flags, don't use `synchronized`—it's heavy. Instead, use the **`java.util.concurrent.atomic`** package (like `AtomicInteger`). It uses low-level CPU instructions to ensure thread safety with much better performance.
